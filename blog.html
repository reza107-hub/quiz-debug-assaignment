1. DIfference between map(), filter(), find(), and forEach() are:

map()

Return a array as the same length of original array with each element based on given function.
Example: 

                          
                            const numbers = [1, 2, 3, 4, 5];
                            const multipliedNumbers = numbers.map((num) => num * 2);
                            console.log(multipliedNumbers); // Output: [2, 4, 6, 8, 10]

filter()

Return an array with elements that match a given condition.
Example: 

                          
                            const numbers = [1, 2, 3, 4, 5];
                          const evenNumbers = numbers.filter((num) => num % 2 === 0);
                          console.log(evenNumbers); // Output: [2, 4]

find()

Return the first element that matches a given condition.
Example: 

                          const numbers = [1, 2, 3, 4, 5];
                            const evenNumber = numbers.find((num) => num % 2 === 0);
                            console.log(evenNumber); // Output: 2

forEach()

forEach does not return anything. It simply executes a provided function once for each element in an array.
Example: 

                          
                            const numbers = [1, 2, 3, 4, 5];
                            numbers.forEach((num) => console.log(num)); // Output: 1 2 3 4 5

2. Difference between localStorage , sessionStorage , cookie are :

localStorage

localStorage is a way store data in web browser. The data stored in localStorage is available in same domain. It doesn't clear the stored data.

sessionStorage

sessionStorage is similar as localStorage, but it cleared stored data when the browser window or tab is closed.

cookie

A cookie is a small piece of data that is stored on the client-side and is sent to the server with every HTTP request. It can store up to 4 KB of data and has an expiration date. The data stored in a cookie is available to all pages from the same domain, including future visits.

3. EventLoop and it's working process :

The event loop is a crucial mechanism in JavaScript that enables it to handle multiple tasks simultaneously without blocking or freezing the main execution thread. Whenever you execute JavaScript code, any long-running or asynchronous tasks, like fetching data from a server or waiting for user input, are added to a task queue. The event loop continually checks this task queue for pending tasks and executes them in the order they are received. It ensures that only one task is processed at a time, and if a task takes too long to complete, other tasks can still execute without being blocked.

4. The working procedure of quizTimer() function :

The quizTimer function is designed to show a timer on our index.html page and update it every second. It also provides an optional dismiss parameter that can be used to stop the timer. The function utilizes the setInterval method to refresh the timer display and track the number of seconds that have passed. Additionally, it formats the minutes and seconds with leading zeros and alternates the timer color every 60 seconds. To stop the timer when needed, clearInterval function is implemented.

5. The working procedure of chooseQuiz() function :

The chooseQuiz function is responsible for handling the selection of a quiz option when a user clicks on it. It accepts two parameters: index and givenAns. The function first checks if an answer already exists for the current quiz. If one exists, it finds the answer's index in the answers array and replaces it with a new object containing the updated givenAns property. In case no answer exists, the function creates a new object by copying the properties of the current quiz object from the quizData array. Once the answers array is updated, the function calls the displayAnswers function to refresh the display of the quiz answers.
